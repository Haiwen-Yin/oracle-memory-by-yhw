## 🔍 Property Graph API Usage

### Method 1: SQL JOIN-based Query (Recommended)

```sql
SELECT 
    n.node_id,
    n.node_type,
    vector_dimension_count(n.embedding) as dims,
    r.relationship_type as rel_type,
    m.target_node_id as connected_to
FROM openclaw.memory_nodes n
LEFT JOIN openclaw.memory_relationships r ON n.node_id = r.source_memory_id
LEFT JOIN openclaw.memory_nodes m ON r.target_memory_id = m.node_id;
```

### Method 2: Property Graph DDL (Schema Definition)

```sql
-- Register property graph for Oracle AI DB
CREATE PROPERTY GRAPH mem_graph
    NODE TABLES (memory_nodes)
        KEY (node_id)
        LABEL node_type
        PROPERTIES (embedding, node_id)
    EDGE TABLES (memory_relationships)
        SOURCE KEY (source_memory_id) REFERENCES memory_nodes(node_id)
        DESTINATION KEY (target_memory_id) REFERENCES memory_nodes(node_id)
        LABEL relationship_type
        PROPERTIES (relationship_type, confidence);
```

### Method 3: GRAPH_TABLE Query (Property Graph Syntax)

```sql
SELECT * FROM GRAPH_TABLE(
    openclaw.mem_graph
    COLUMNS (n.node_id, n.node_type, vector_dimension_count(n.embedding)) as dims
    NODES MATCH (n:memory_nodes)
    WHERE n.node_id IN (10023, 10024, 10025)
);
```

## ⚠️ Common Issues & Solutions

### Issue 1: Foreign Key Constraint Violation

**Error:** `ORA-02291: integrity constraint violated - parent key not found`

**Solution:** Drop foreign key constraints before inserting test data
```sql
-- Check FK constraints
SELECT constraint_name FROM user_constraints 
WHERE table_name = 'MEMORY_RELATIONSHIPS' AND constraint_type = 'R';

-- Drop each FK constraint
ALTER TABLE openclaw.memory_relationships DROP CONSTRAINT SYS_C008630;
ALTER TABLE openclaw.memory_relationships DROP CONSTRAINT SYS_C008631;
```

### Issue 2: Identity Column Insertion Error

**Error:** `ORA-32795: cannot insert into a generated always identity column`

**Solution:** Omit the ID column in INSERT statements (it's auto-generated)
```sql
-- ❌ WRONG - includes ID
INSERT INTO memory_relationships (id, source_memory_id, ...) VALUES (...);

-- ✅ CORRECT - omit ID
INSERT INTO memory_relationships (source_memory_id, target_memory_id, ...) VALUES (...);
```

### Issue 3: Property Graph Column Names

**Note:** Property Graph edge tables use `source_memory_id` / `target_memory_id`, NOT `source_node_id` / `dest_node_id`.

The HNSW index on `embedding` column enables fast vector similarity search:

1. Primary key indexes on `node_id`, `source_memory_id`, `target_memory_id`
